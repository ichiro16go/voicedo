# ADR 0002: 記事生成 LLM の選定（ブラインドテスト結果に基づく確定）

- 状態: Accepted (2026-06-23)
- 関連: [tech-stack.md](../tech-stack.md), [plan.md](../plan.md), [blind-test-results.md](../blind-test-results.md)

## 決定
音声日記アプリ **Voicedo** の記事生成 LLM として、**Claude 3.5 Sonnet (claude-3-5-sonnet-20241022)** を正本採用する。

## 文脈
- 音声日記（3-5分の不完全で口語的な書き起こしテキスト）から、読者に響く高品質な note 記事（ハル：エントリ、エッセイ風）や Qiita 記事（ロイ：技術ログ）を自動生成したい。
- 記事生成における以下の重要な制約（インバリアント）を確実に守らせたい：
  - **ハル**: 文字数 700〜1200字、敬体、{{REDACT}}による固有名詞の匿名化、末尾の `<!-- redactions -->` コメントへの記録、Markdown のみ出力（前置き・挨拶の禁止）。
  - **ロイ**: 文字数 800〜1500字、常体、特定の構造、推測による技術情報補完の禁止、曖昧な箇所への `(要確認)` 付与。
- これらを満たす最適なプロバイダー/モデルを決定するため、3 社の最高峰モデル（Model A: Claude 3.5 Sonnet, Model B: GPT-4o, Model C: Gemini 1.5 Pro）を対象に、実音声をシミュレートした 5 本のテストトークを用いたブラインドテストを実装・実施した。

## 結果
- **検証での発見**:
  - **Claude 3.5 Sonnet (Model A)** は、すべての要件（REDACT 処理、HTML コメント記録、(要確認) 表記の付与、Markdown のみ出力）を完全にノーエラーで順守した唯一のモデルであった。文章の自然さ、情緒的な表現、タイトル選定センスも群を抜いており、本アプリのコア価値に最も合致した。
  - **GPT-4o (Model B)** は、構成等は手堅いものの、文章がビジネス風で均一（やや冷たい/AIっぽい）であり、また `(要確認)` などの細かい要件漏れや推測補正の違反が見られた。
  - **Gemini 1.5 Pro (Model C)** は、おしゃべりな前置き（お礼や要約以外の解説）を削りきれない指示順守上の課題が見られ、本番システムに流す際のエラーハンドリングコストが高くなると判定された。
- **採用方針**:
  - コア体験である記事生成フェーズには、日本語品質・要件順守率ともに圧倒的最優秀であった **Claude 3.5 Sonnet** を採用。
  - プロバイダーアダプタ（`src/lib/ai/`）は、将来的なモデル切り替え（gpt-5 / Claude 4.5 への進化など）に備え、マルチプロバイダーの抽象化レイヤーを維持する。
